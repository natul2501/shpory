HTTP (англ. HyperText Transfer Protocol) – основной протокол передачи гипертекста, использующийся для транспортировки данных сайтов. Действие этого стандарта основано на клиент-серверной технологии: клиент отправляет запрос на сервер, который выполняет его и высылает результат. Подробнее: https://cityhost.ua/blog/chto-takoe-https.html

ARP (англ. Address Resolution Protocol — протокол определения адреса) — протокол в компьютерных сетях, предназначенный для определения MAC-адреса по IP-адресу другого компьютера.

TCP — это сложный, требующий больших затрат времени протокол, что объясняется его механизмом установления соединения, но он берет на себя заботу о гарантированной доставке пакетов, избавляя нас от необходимости включать эту функциональную возможность в прикладной протокол.

UDP (англ. User Datagram Protocol — протокол пользовательских датаграмм) - в отличие от TCP, очень быстрый протокол, поскольку в нем определен самый минимальный механизм, необходимый для передачи данных. С UDP компьютерные приложения могут посылать сообщения (в данном случае называемые датаграммами) другим хостам по IP-сети без необходимости предварительного сообщения для установки специальных каналов передачи или путей данных.
